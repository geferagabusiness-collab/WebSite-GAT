# GITHUB WORKFLOW

# Repositorio Oficial

Todo el proyecto debe mantenerse sincronizado con GitHub.

---

# Flujo

1. Pull latest.
2. Crear branch.
3. Implementar.
4. Probar.
5. Commit.
6. Push.
7. Merge.

---

# Formato Commit

```txt
feat(hero): cinematic homepage animation
fix(navbar): responsive overlap issue
refactor(alexandro): optimize hologram loader
```

---

# Regla

Nunca subir:

- node_modules,
- logs,
- builds,
- secretos,
- .env.
